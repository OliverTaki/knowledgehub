Several animation source notes are useful for the same reason: they preserve a behavior that can be studied later, not only a clip to admire. For Knowledge Hub, the reusable unit is the motion problem: handmade timing, puppet weight, soft-ground deformation, camera handling, surreal advertising rhythm, or AI-generated painterly movement.

Motion references need a retrieval reason

The Ponkobo stop-motion car note is not only a charming toy-car clip. Its archival value is in the tiny performance decisions: eyelid timing, steering movement, material texture, and how handmade objects keep weight visible. Those are the details a future animator, rigger, or motion designer might need to retrieve.

Handmade clips and AI clips should be filed differently

The impressionist AI animation source is useful as a debate and style reference, while the Jump puppet commercial, Mirai Mizue short, and Wildwood production note are useful as craft references. Keeping those roles separate prevents the archive from flattening every moving image into the same tag. Some sources help with aesthetic direction; others help with timing, tactile staging, or production method.

Virtual cameras are also motion references

The Jonathan Winbush Rokoko, Cinema 4D, and Unreal source note belongs beside animation references because the reusable element is not just a tool chain. It is the camera behavior: handheld framing, embodied operation, motion-capture data, and real-time scene review all shape the final feeling of movement.

What to preserve in Knowledge Hub

Future animation Wire entries should record the movement behavior first: weight, delay, deformation, camera agency, puppet material, loop rhythm, or generative artifact. The source URL still matters, but the retrieval reason is what lets a saved clip become production memory.
